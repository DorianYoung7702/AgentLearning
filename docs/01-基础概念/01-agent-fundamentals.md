# Agent 基础概念与架构

## 1. 什么是 AI Agent？

**AI Agent（智能体）** 是一个能够**感知环境、自主决策、执行行动**以达成特定目标的系统。与传统的 LLM 对话不同，Agent 具备：

- **自主性**：能独立规划和执行多步任务
- **工具使用能力**：可调用外部工具（API、数据库、代码执行等）
- **记忆**：能记住历史交互，维护上下文
- **推理能力**：能分解复杂问题、反思和修正

### Agent vs. ChatBot 区别

| 维度 | ChatBot | Agent |
|------|---------|-------|
| 交互模式 | 单轮/多轮对话 | 自主多步执行 |
| 能力边界 | 仅文本生成 | 文本 + 工具调用 + 行动 |
| 决策方式 | 被动响应 | 主动规划 |
| 目标 | 回答问题 | 完成任务 |

## 2. Agent 核心架构

### 2.1 经典架构：感知-规划-行动（Perception-Planning-Action）

```
用户输入 → [感知模块] → [规划模块] → [行动模块] → 环境反馈
                ↑                                      |
                └──────────────────────────────────────┘
```

- **感知（Perception）**：接收并理解用户输入、环境状态、工具返回结果
- **规划（Planning）**：分解任务、制定执行计划、选择工具
- **行动（Action）**：执行具体操作（调用 API、生成代码、搜索信息等）
- **反馈循环**：根据行动结果调整后续计划

### 2.2 LLM-based Agent 通用架构

现代 Agent 以 LLM 为"大脑"，典型组件包括：

```
┌─────────────────────────────────────────┐
│              AI Agent                    │
│                                          │
│  ┌──────────┐  ┌────────────┐           │
│  │  LLM     │  │  记忆系统   │           │
│  │ (大脑)   │←→│ (Memory)   │           │
│  └────┬─────┘  └────────────┘           │
│       │                                  │
│  ┌────┴─────┐  ┌────────────┐           │
│  │  规划器   │  │  工具集     │           │
│  │(Planner) │→→│ (Tools)    │           │
│  └──────────┘  └────────────┘           │
└─────────────────────────────────────────┘
```

四大核心组件：
1. **LLM（大脑）**：负责理解、推理、决策
2. **记忆系统（Memory）**：短期记忆（对话上下文）+ 长期记忆（向量存储）
3. **规划器（Planner）**：任务分解、步骤编排
4. **工具集（Tools）**：可调用的外部能力（搜索、代码执行、API 等）

## 3. Agent 的工作流程

### 3.1 ReAct 范式（Reasoning + Acting）

最经典的 Agent 运行范式，交替进行**推理**和**行动**：

```
循环：
  1. Thought（思考）：分析当前状态，决定下一步
  2. Action（行动）：选择并执行一个工具
  3. Observation（观察）：获取工具返回结果
  → 回到步骤1，直到任务完成
最终：生成 Final Answer
```

**示例**：
```
用户：北京今天天气怎么样？适合户外活动吗？

Thought: 我需要查询北京今天的天气信息
Action: search_weather(city="北京")
Observation: 北京今天晴，25°C，微风

Thought: 天气晴朗温度适宜，可以给出建议了
Final Answer: 北京今天天气晴朗，气温25°C，非常适合户外活动！
```

### 3.2 Plan-and-Execute 范式

先整体规划，再逐步执行：

```
1. 接收任务
2. 制定完整计划（拆分为子任务列表）
3. 逐步执行每个子任务
4. 执行完一步后，评估是否需要调整计划
5. 所有子任务完成后，汇总结果
```

### 3.3 两种范式对比

| 维度 | ReAct | Plan-and-Execute |
|------|-------|-----------------|
| 规划方式 | 逐步规划 | 先整体规划 |
| 灵活性 | 高，每步可调整 | 中，需要重规划机制 |
| 适用场景 | 探索性任务 | 结构明确的复杂任务 |
| 效率 | 可能有冗余步骤 | 执行路径更清晰 |

## 4. Agent 类型分类

### 按自主程度分类
- **L1 - 路由型**：根据意图路由到不同处理流程（最简单）
- **L2 - 工具调用型**：能使用工具完成特定任务
- **L3 - 自主规划型**：能自主分解任务并规划执行
- **L4 - 多 Agent 协作型**：多个 Agent 协同完成复杂任务

### 按应用场景分类
- **对话型 Agent**：客服、助手
- **任务型 Agent**：自动化工作流
- **创作型 Agent**：内容生成
- **研究型 Agent**：信息收集与分析
- **编程型 Agent**：代码生成与调试（如 Cursor、Windsurf）

## 5. 面试高频问题

### Q1: 请解释什么是 AI Agent，它和传统 LLM 应用有什么区别？
**要点**：Agent = LLM + 记忆 + 工具 + 规划。传统 LLM 应用是被动的文本生成，Agent 是主动的任务执行者，具有自主决策和行动能力。

### Q2: Agent 的核心组件有哪些？各自的作用是什么？
**要点**：四大组件 — LLM（推理决策）、Memory（上下文维护）、Tools（外部能力）、Planner（任务分解）。

### Q3: 请描述 ReAct 范式的工作流程。
**要点**：Thought → Action → Observation 循环。LLM 交替进行推理和行动，通过观察结果指导下一步，直到得出最终答案。

### Q4: Agent 设计中最大的挑战是什么？
**要点**：
- 可靠性：LLM 可能产生幻觉导致错误决策
- 规划质量：复杂任务的分解可能不合理
- 错误恢复：行动失败后的重试和回退策略
- 成本控制：多步调用导致的 Token 消耗
- 安全性：工具调用的权限控制
