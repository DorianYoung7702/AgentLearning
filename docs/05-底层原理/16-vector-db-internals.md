# 向量数据库原理

> 向量数据库是 RAG 和 Agent 记忆系统的基础设施，大厂面试会考察底层索引算法。

## 1. 向量检索的核心问题

给定一个查询向量 q，在 N 个高维向量中找到最近的 K 个向量。

**暴力搜索（Brute Force）**：
- 计算 q 与所有 N 个向量的距离，取 Top-K
- 时间复杂度：O(N × d)，d 是维度
- 当 N = 百万级时，速度不可接受

**近似最近邻搜索（ANN）**：
- 牺牲少量精度，换取大幅加速
- 常用 Recall@K > 95% 即可满足需求

## 2. 距离度量

### 2.1 常用距离

| 距离 | 公式 | 特点 |
|------|------|------|
| **余弦相似度** | cos(A,B) = A·B / (\|\|A\|\| × \|\|B\|\|) | 最常用，衡量方向 |
| **欧氏距离（L2）** | \|\|A-B\|\|₂ | 衡量绝对距离 |
| **点积（IP）** | A·B | 归一化后等价于余弦 |

### 2.2 选择建议

- **已归一化的向量**：三种等价，用点积最快
- **未归一化**：用余弦相似度（自动归一化）
- **需要考虑向量模长**：用欧氏距离

## 3. 索引算法

### 3.1 IVF（Inverted File Index）

**原理**：将向量空间聚类，查询时只搜索最近的几个聚类。

```
索引构建：
  1. 用 K-Means 将 N 个向量聚为 C 个簇
  2. 每个向量归入最近的簇

查询：
  1. 计算 query 与所有簇中心的距离
  2. 选择最近的 nprobe 个簇
  3. 只在这些簇内暴力搜索

            C1(搜索) C2      C3(搜索) C4
           / \       |      / \       |
          v1 v2     v3    v4 v5     v6
```

| 参数 | 说明 |
|------|------|
| nlist (C) | 聚类数量，通常 √N |
| nprobe | 查询时搜索的簇数，越大越精确但越慢 |

**优缺点**：
- ✅ 实现简单，效果好
- ✅ 适合更新不频繁的数据
- ❌ 聚类边界附近的向量可能被遗漏

### 3.2 HNSW（Hierarchical Navigable Small World）

**原理**：构建多层图结构，从高层稀疏图快速定位到低层密集图中的近邻。

```
Layer 2 (最稀疏):  A ─────────── D
                          
Layer 1 (中等):    A ──── C ──── D
                     \   |
Layer 0 (最密集):  A ─ B ─ C ─ E ─ D ─ F

查询流程：
  1. 从最高层的入口节点开始
  2. 在每层中贪婪搜索最近邻
  3. 找到后进入下一层继续搜索
  4. 在 Layer 0 中精确定位 Top-K
```

| 参数 | 说明 | 典型值 |
|------|------|--------|
| M | 每个节点的连接数 | 16-64 |
| ef_construction | 构建时的搜索范围 | 200 |
| ef_search | 查询时的搜索范围 | 50-200 |

**优缺点**：
- ✅ 查询速度极快（对数复杂度）
- ✅ 精度高（Recall > 99%）
- ✅ 支持动态插入
- ❌ 内存开销大（需要存储图结构）
- ❌ 构建时间较长

### 3.3 PQ（Product Quantization，乘积量化）

**原理**：将高维向量压缩为短码，减少内存和加速距离计算。

```
原始向量 (128维):
  [0.12, -0.34, ..., 0.78]  → 128 × 4 bytes = 512 bytes

PQ 压缩：
  1. 将 128 维分成 8 个子空间，每个 16 维
  2. 在每个子空间中用 256 个聚类中心（码本）
  3. 每个子向量用 1 字节的码本 ID 表示

压缩后: [23, 156, 87, 42, 201, 11, 99, 178] → 8 bytes

压缩比: 512 / 8 = 64 倍
```

**优缺点**：
- ✅ 极大压缩内存
- ✅ 可与 IVF 结合使用（IVF-PQ）
- ❌ 精度有损

### 3.4 索引算法对比

| 算法 | 查询速度 | 精度 | 内存 | 构建速度 | 动态更新 |
|------|---------|------|------|---------|---------|
| **Flat（暴力）** | 最慢 | 100% | 基线 | 最快 | ✅ |
| **IVF** | 快 | 高 | 基线 | 中 | ❌需重建 |
| **HNSW** | 最快 | 最高 | 最大 | 慢 | ✅ |
| **PQ** | 快 | 中 | 最小 | 中 | ❌ |
| **IVF-PQ** | 快 | 中高 | 小 | 中 | ❌ |
| **HNSW+PQ** | 快 | 高 | 中 | 慢 | ✅ |

### 3.5 选择建议

| 场景 | 推荐 |
|------|------|
| 小规模（< 10万） | Flat 或 HNSW |
| 中规模（10万-千万）| HNSW |
| 大规模（千万+）| IVF-PQ 或 HNSW+PQ |
| 内存受限 | IVF-PQ |
| 高精度要求 | HNSW |
| 需要频繁更新 | HNSW |

## 4. 向量数据库架构

### 4.1 核心组件

```
┌──────────────────────────────────┐
│         向量数据库                │
│                                  │
│  ┌────────────┐  ┌────────────┐ │
│  │ 索引引擎    │  │ 存储引擎    │ │
│  │ (ANN Index) │  │ (向量+元数据)│ │
│  └────────────┘  └────────────┘ │
│                                  │
│  ┌────────────┐  ┌────────────┐ │
│  │ 查询引擎    │  │ 过滤引擎    │ │
│  │ (检索+排序) │  │ (元数据过滤)│ │
│  └────────────┘  └────────────┘ │
└──────────────────────────────────┘
```

### 4.2 元数据过滤

```
查询时不仅按向量相似度，还要按元数据条件过滤：

query = {
    "vector": [0.1, 0.2, ...],
    "filter": {
        "department": "engineering",
        "date": {"$gte": "2024-01-01"}
    },
    "top_k": 5
}
```

**过滤策略**：
- **Pre-filtering**：先过滤再搜索（精确但可能太少结果）
- **Post-filtering**：先搜索再过滤（可能浪费计算）
- **Hybrid**：结合两者

## 5. Embedding 模型

### 5.1 模型架构

```
典型 Embedding 模型：

文本 → [Tokenizer] → [Transformer Encoder] → [Pooling] → 向量

Pooling 策略：
  - CLS Token: 取 [CLS] 位置的输出
  - Mean Pooling: 所有 Token 输出的均值（更常用）
  - Last Token: 最后一个 Token 的输出
```

### 5.2 训练方法

| 方法 | 说明 |
|------|------|
| **对比学习** | 正例对拉近、负例对推远 |
| **Hard Negative Mining** | 使用困难负例提升模型 |
| **多阶段训练** | 弱监督预训练 → 有监督微调 |
| **知识蒸馏** | 用大模型指导小模型 |

### 5.3 评估指标

- **MTEB Benchmark**：最权威的 Embedding 评估基准
- 覆盖：检索、分类、聚类、重排等任务

## 6. 面试高频问题

### Q1: 解释 HNSW 的工作原理。
**要点**：构建多层小世界图，高层稀疏低层密集。查询时从顶层入口开始，每层贪婪搜索最近邻，逐层下降到最底层得到精确结果。参数 M 控制连接数，ef_search 控制搜索范围。时间复杂度 O(log N)。

### Q2: IVF 和 HNSW 如何选择？
**要点**：HNSW 查询最快、精度最高，但内存大；IVF 内存友好但精度略低且不支持动态更新。中小规模选 HNSW，大规模+内存受限选 IVF-PQ。需要频繁更新选 HNSW。

### Q3: 什么是乘积量化（PQ）？
**要点**：将高维向量分成多个子空间，每个子空间用聚类中心的 ID 代替原始向量。可实现 32-128 倍的压缩比。常与 IVF 结合使用（IVF-PQ）。

### Q4: 向量检索时如何做元数据过滤？
**要点**：Pre-filtering（先过滤再搜索，精确但可能结果少）、Post-filtering（先搜索再过滤，可能浪费计算）、Hybrid（结合两者）。选择取决于过滤条件的选择率。

### Q5: 如何评估向量检索的效果？
**要点**：Recall@K（Top-K 中包含正确结果的比例）、QPS（每秒查询数）、延迟（P99）、内存占用。生产中通常要求 Recall@10 > 95%。

### Q6: 余弦相似度和欧氏距离有什么区别？什么时候用哪个？
**要点**：余弦衡量方向（角度），对向量长度不敏感；欧氏距离衡量绝对距离。Embedding 已归一化时两者等价。NLP/语义搜索通常用余弦相似度。
