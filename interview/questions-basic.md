# 基础面试题集

> 涵盖 Agent 基础概念、LLM 原理、Prompt 工程等基础知识。每题附参考答案要点。

---

## 一、Agent 基础概念

### Q1: 什么是 AI Agent？它与传统 ChatBot 有什么区别？

**参考要点**：
- Agent = 能感知环境、自主决策、执行行动的智能系统
- Agent = LLM（大脑）+ Memory（记忆）+ Tools（工具）+ Planner（规划）
- ChatBot 被动响应文本，Agent 主动规划和执行任务
- Agent 具有自主性、工具使用能力、记忆和推理能力

### Q2: 描述 Agent 的核心组件及各自作用。

**参考要点**：
1. **LLM（大脑）**：理解、推理、决策的核心
2. **Memory（记忆）**：短期记忆（对话上下文）+ 长期记忆（持久化存储）
3. **Tools（工具）**：扩展 Agent 能力的外部接口（搜索、计算、API等）
4. **Planner（规划器）**：任务分解、步骤编排、执行策略选择

### Q3: 什么是 ReAct？请举例说明。

**参考要点**：
- ReAct = Reasoning + Acting，推理和行动交替进行
- 流程：Thought → Action → Observation → Thought → ... → Final Answer
- 例子：用户问"今天适合出门吗？"→ Thought: 需要查天气 → Action: search_weather("北京") → Observation: 晴，25°C → Thought: 天气好 → Final Answer: 适合出门

### Q4: Agent 设计中的主要挑战有哪些？

**参考要点**：
- **可靠性**：LLM 幻觉导致错误决策
- **规划质量**：复杂任务分解可能不合理
- **错误恢复**：行动失败后的重试和回退
- **成本控制**：多步调用的 Token 消耗
- **安全性**：工具调用权限控制、Prompt Injection

### Q5: Agent 有哪些分类方式？

**参考要点**：
- 按自主程度：路由型 → 工具调用型 → 自主规划型 → 多Agent协作型
- 按应用：对话型、任务型、创作型、研究型、编程型
- 按架构：单Agent、多Agent、人机协作

## 二、大语言模型（LLM）

### Q6: 简述 Transformer 的自注意力机制。

**参考要点**：
- 输入通过 W_Q, W_K, W_V 三个矩阵分别投影得到 Q, K, V
- 注意力分数 = softmax(Q·K^T / √d_k) · V
- √d_k 缩放防止点积值过大导致梯度消失
- 多头注意力让不同头关注不同类型的信息

### Q7: Decoder-Only 架构为什么成为主流？

**参考要点**：
- 自回归生成天然适合文本生成任务
- 训练目标简单统一（Next Token Prediction）
- Scaling Law 验证其扩展性好
- 通过 In-Context Learning 可处理各种任务，无需 Encoder

### Q8: Temperature 和 Top-p 参数的作用是什么？Agent 中如何设置？

**参考要点**：
- Temperature 控制输出随机性：0=确定性输出，1=更随机
- Top-p（核采样）：从累积概率前 p 的 Token 中采样
- Agent 场景通常用低温度（0~0.3），因为需要可靠一致的输出
- 工具调用格式解析要求确定性输出

### Q9: 什么是 RLHF？对比 RLHF 和 DPO。

**参考要点**：
- RLHF = 基于人类反馈的强化学习：训练奖励模型 → PPO 优化策略
- DPO = 直接偏好优化：直接用偏好对训练，无需奖励模型
- RLHF 训练复杂但经验成熟；DPO 实现简单但可能在复杂场景效果不如

### Q10: LLM 有哪些局限性？Agent 中如何应对？

**参考要点**：
| 局限性 | 应对 |
|--------|------|
| 幻觉 | RAG + 工具验证 |
| 知识截止 | 搜索工具 |
| 推理不稳定 | CoT + Self-consistency |
| 上下文有限 | 记忆管理 + 摘要压缩 |
| 无法行动 | 工具调用 |

## 三、提示工程

### Q11: System Prompt 应该包含哪些关键要素？

**参考要点**：
1. 角色定义（身份、专业领域）
2. 能力和限制说明
3. 可用工具列表及详细描述
4. 输出格式要求
5. 行为约束和安全规则
6. 错误处理指引

### Q12: 对比 Zero-shot、Few-shot 和 CoT。

**参考要点**：
- **Zero-shot**：无示例直接给指令，简单但效果不稳定
- **Few-shot**：提供示例引导输出格式，消耗更多Token但更可靠
- **CoT**：引导逐步推理，提升复杂任务的准确率
- Agent 中：工具调用常用 Few-shot 确保格式正确；推理用 CoT

### Q13: Prompt 优化的流程是什么？

**参考要点**：
1. 编写初版 Prompt
2. 构建测试用例集
3. 运行测试，分析失败案例
4. 针对性修改 Prompt
5. 回归测试（确保不破坏已有能力）
6. 迭代重复

### Q14: 什么是 Structured Output？为什么在 Agent 中很重要？

**参考要点**：
- 让 LLM 输出结构化数据（如 JSON）
- Agent 需要从 LLM 输出中解析出工具调用的函数名和参数
- 如果输出格式不正确，程序无法解析，Agent 就无法执行
- 通过 System Prompt 约定格式 + Few-shot 示例 + JSON Mode 保证

### Q15: 常见的 Prompt 陷阱有哪些？

**参考要点**：
- 指令模糊导致理解偏差
- 角色冲突（多个矛盾要求）
- 格式不一致导致解析失败
- 上下文过长关键信息被忽略
- 缺少边界条件和异常处理说明

## 四、工具使用

### Q16: 解释 Function Calling 的完整流程。

**参考要点**：
1. 开发者定义工具（JSON Schema 描述函数名、参数、描述）
2. 用户发送请求
3. LLM 判断是否需要调用工具
4. 如需要，LLM 生成工具调用 JSON（函数名 + 参数）
5. 应用层解析并执行工具调用
6. 将工具结果返回给 LLM
7. LLM 基于结果生成最终回答

### Q17: 如何设计一个好的工具？

**参考要点**：
- 职责单一（一个工具只做一件事）
- 命名清晰（`search_company_docs` 优于 `search`）
- 参数类型、范围、必填/选填要明确
- 描述详尽（用途、适用场景、不适用场景、返回格式）
- 有意义的错误信息
- 幂等性（查询类工具应保证）

### Q18: Agent 工具调用失败了怎么办？

**参考要点**：
- 区分错误类型：参数错误 / 工具不可用 / 超时 / 结果为空
- 参数错误：让 LLM 修正参数重试
- 工具不可用：使用替代工具或直接回答
- 设置最大重试次数（2-3次）
- 必要时降级为直接回答或请求用户帮助

### Q19: 工具调用的安全风险和应对措施？

**参考要点**：
- 风险分级：只读（低） → 数据写入（中） → 系统操作（高）
- Human-in-the-loop：高危操作需用户确认
- 沙箱执行代码
- 参数验证
- 速率限制
- 审计日志

### Q20: 并行工具调用和串行调用如何选择？

**参考要点**：
- 并行：无依赖关系的工具（如同时查多个城市天气），减少延迟
- 串行：有依赖关系（如先搜索再基于结果计算）
- 注意并行调用中部分失败的处理

## 五、记忆系统

### Q21: Agent 的短期记忆和长期记忆有什么区别？

**参考要点**：
| | 短期记忆 | 长期记忆 |
|---|---------|---------|
| 实现 | LLM 上下文窗口 | 向量DB/关系DB |
| 容量 | 受 Token 限制 | 理论无限 |
| 读写 | 自动 | 显式 |
| 持久性 | 会话内 | 跨会话 |

### Q22: 上下文窗口不够用时怎么办？

**参考要点**：
- 滑动窗口裁剪旧对话
- 摘要压缩历史
- 关键信息存入长期记忆按需检索
- 分配上下文预算（System Prompt / 记忆 / 历史 / 输出）
- 工具结果截断

### Q23: 什么是 Embedding？向量检索的工作原理？

**参考要点**：
- Embedding：将文本转为高维向量，语义相近的文本向量接近
- 流程：文本 → Embedding 模型 → 向量 → 存入向量DB
- 检索：Query → Embedding → 在向量空间找 Top-K 最近邻
- 距离度量：余弦相似度最常用

### Q24: 什么是 "Lost in the Middle" 问题？

**参考要点**：
- LLM 对长上下文中间部分的注意力低于开头和结尾
- 导致中间的重要信息被忽略
- 应对：重要信息放开头/结尾、控制上下文长度、Rerank 优化排列

### Q25: 如何设计 Agent 的记忆管理策略？

**参考要点**：
1. 区分短期/长期记忆的存储
2. 定义写入触发条件（用户要求/重要信息检测/会话结束总结）
3. 设计检索策略（语义+关键词混合搜索）
4. 上下文预算分配
5. 记忆更新和冲突处理
6. 过期淘汰策略
